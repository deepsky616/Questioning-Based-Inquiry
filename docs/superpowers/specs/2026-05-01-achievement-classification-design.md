# 성취기준 분류 선택 및 전 교과 단원 데이터 설계

**날짜**: 2026-05-01  
**범위**: `teacher-curriculum/page.tsx` UI 개선 + 전 교과 단원 시드 데이터

---

## 문제 정의

- `curriculum_areas.units` JSONB 컬럼에 단원 목록 `[{unitCode, unitName}]`이 저장됨
- 과학 일부 학년군만 데이터가 있고, 나머지 교과는 `[]` → 단원 분류 기능 미작동
- "단원 선택" 섹션과 "성취기준" 섹션이 분리되어 있어 UX가 어색함

---

## 목표

1. **전 교과** (국어·수학·사회·도덕·과학·체육·음악·미술·영어·실과·바른생활·슬기로운생활·즐거운생활) 모든 학년군·영역에 2022 개정 교육과정 기준 단원명 데이터 추가
2. "단원 선택"과 "성취기준" 두 섹션을 **하나의 카드**로 통합하여 UX 개선
3. Step 2(핵심어 선택)가 최종 선택된 성취기준 기반으로 작동하는 흐름 유지

---

## 역할 분담

### Codex 역할 (웹 리서치)
- 2022 개정 교육과정 고시 문서에서 전 교과 × 전 학년군 × 전 영역의 단원명 수집
- 출력 형식: SQL UPDATE 시드 스크립트 (`scripts/seed-units.sql`)
- 각 행은 `curriculum_areas.id`를 기준으로 `units` 컬럼을 업데이트

### Claude Code 역할 (구현)
- `teacher-curriculum/page.tsx` UI 통합 개선
- 시드 스크립트 생성 지원 및 검증
- 기존 `getFilteredAchievements()` → `callGenerate("keywords")` 흐름 유지

---

## UI 설계

### 현재 구조
```
[단원 선택 섹션]   ← 분리됨
[성취기준 섹션]    ← 분리됨
```

### 변경 후 구조
```
[성취기준 선택 섹션 — 통합]
  ┌─────────────────────────────────────────────────────┐
  │ 단원(분류) 선택                    [전체 선택] [전체 해제] │
  │                                                     │
  │ [지층과 화석 ✓] [빛의 성질 ✓] [용해와 용액] ...      │
  │                                                     │
  │ ── 선택된 단원의 성취기준 ──────────────────────────  │
  │ [지층과 화석]                                        │
  │   · [4과02-01] 퇴적암이 만들어지는 과정을...          │
  │   · [4과02-02] 화석이 만들어지는 과정을...            │
  │ [빛의 성질]                                          │
  │   · [4과03-01] 빛이 직진함을 알고...                  │
  │                                                     │
  │ 2개 단원 선택됨 · 4개 성취기준 적용                   │
  └─────────────────────────────────────────────────────┘
```

### 동작 규칙
- 단원 데이터 있는 교과: 단원 칩 다중 선택/해제 → 해당 성취기준 즉시 그룹 표시
- 단원 데이터 없는 교과: 전체 성취기준 평면 표시 (현행 유지)
- "다음 단계" 버튼: 단원 데이터 있을 때 최소 1개 단원 선택 필수

---

## 데이터 설계

### `curriculum_areas.units` 컬럼 형식 (기존 유지)
```json
[
  { "unitCode": "01", "unitName": "지층과 화석" },
  { "unitCode": "02", "unitName": "화산과 지진" }
]
```

### 성취기준 코드와 단원 매핑 규칙 (기존 유지)
- 코드 `[4과02-01]` → unitCode `"02"` 추출 (`extractUnitCode` 함수 재사용)
- 코드 형식이 다른 교과(국어 `[4국01-01]` 등)도 동일 패턴 적용

### 시드 스크립트 위치
`scripts/seed-units.sql` — Codex가 생성, SQL UPDATE 형식

- 단원 데이터가 없는 영역은 기존처럼 전체 성취기준 표시 (하위 호환)

- 전 교과 동일한 UI 패턴 적용
